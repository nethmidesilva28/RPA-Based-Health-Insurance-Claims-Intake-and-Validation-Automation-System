# Claim Validation Logic

This section defines the business rules applied during claim validation.

## 1. Policy Validation Rules
- Policy number must exist in the system
- Policy must be active on the incident date
- Policy must cover the claim type

## 2. Claim Data Validation
- Claimant name must not be empty
- Incident date must be valid and not in the future
- Claim type must be recognized

## 3. Document Validation Rules
- Medical claims require:
  - Hospital invoice
  - Medical report/discharge summary
- Emergency claims may require additional supporting documents
- Missing mandatory documents = Incomplete claim

## 4. Duplicate Check
- Claims with same:
  - Policy number
  - Incident date
  - Claim type
  are flagged as potential duplicates

## 5. Validation Outcome Logic
- If all rules pass → Mark as VALID
- If documents missing → Mark as INCOMPLETE
- If policy invalid → Mark as EXCEPTION
