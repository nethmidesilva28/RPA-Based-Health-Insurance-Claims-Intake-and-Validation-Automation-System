# Future State Overview (To-Be Process)

The proposed future state introduces RPA to automate the claims intake and validation process.

## Future Workflow:

1. Claim Intake Automation
   - Bot captures claims from email/forms automatically

2. Data Extraction
   - Bot extracts key claim details using structured rules/OCR

3. Policy Validation
   - Bot checks policy validity and status in internal systems

4. Document Check
   - Bot verifies required documents based on claim type

5. Rule-Based Validation
   - Bot applies business rules to determine claim completeness

6. Claim Classification
   - Bot categorizes claims as:
     - Valid
     - Incomplete
     - Exception

7. Automated Routing
   - Valid claims → Claims assessment queue
   - Incomplete claims → Pending documents queue
   - Exceptions → Manual review queue

8. Notifications
   - Automated emails sent for missing information or confirmation

9. Audit Logging
   - All bot actions are logged for traceability

## Key Improvements:
- Reduced manual workload
- Faster claim processing
- Standardized validation process
- Improved accuracy and consistency
