# MVP Scope (Minimum Viable Product)

This document defines the minimum set of features required to deliver the first usable version of the RPA-based Health Insurance Claims Intake and Validation system.

## Objective of MVP
The MVP focuses on automating the core claims intake and validation process to reduce manual workload and improve processing speed.

---

## In Scope (MVP Features)

### 1. Claim Intake Automation
- Capture incoming claims from email or submission forms
- Extract key claim details (policy number, claimant name, claim type, incident date)

### 2. Policy Validation
- Validate policy existence in the system
- Check whether the policy is active during incident date

### 3. Document Verification
- Check for mandatory supporting documents based on claim type

### 4. Claim Classification
- Classify claims into:
  - Valid
  - Incomplete
  - Exception

### 5. Routing Logic
- Route claims to appropriate queues:
  - Valid Claims Queue
  - Incomplete Claims Queue
  - Exception Queue

### 6. Basic Notifications
- Send email notifications for missing documents or claim status updates

---

## Out of Scope (MVP)

- Fraud detection logic
- Advanced analytics or AI decision-making
- Claim settlement or payment processing
- External hospital system integrations beyond basic validation
